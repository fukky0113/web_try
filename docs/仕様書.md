## 仕様書

### 目的
- Next.js（App Router）とTailwind CSSで、複数画面を持つ学習アプリケーションを構築する。
- Server Actions を活用し、APIルートを作らずにサーバー処理を実現する。

### ルーティング
| パス | 画面名 | 種別 |
|------|--------|------|
| `/` | タスク管理ダッシュボード | Client Component |
| `/vocabulary` | TOEIC単語帳アプリ | Server + Client Component |
| `/quiz` | クイズ画面（学習機能） | Server + Client Component |

### 画面構成

#### 1. タスク管理ダッシュボード（`/`）
- ヘッダー
  - タイトル「AIX Task Dashboard」
  - 未完了タスク件数
  - 「単語帳」「クイズ」への遷移ボタン
- メインセクション
  - タスク入力フォーム + 追加ボタン
  - タスク一覧（チェックボックス、削除ボタン付き）
  - データ管理: `useState`（メモリ上）

#### 2. TOEIC単語帳アプリ（`/vocabulary`）
- ヘッダー
  - タイトル「TOEIC Vocabulary」
  - 登録単語件数
  - 「ホーム」「クイズ」への遷移ボタン
- メインセクション
  - 単語（英語）と意味（日本語）の入力フォーム
  - Server Actionsでデータ追加（API不要）
  - 単語一覧（削除ボタン付き）
  - データ管理: サーバー側メモリ（Server Actions）

#### 3. クイズ画面（`/quiz`）
- ヘッダー
  - タイトル「TOEIC Quiz」
  - 出題数
  - 「単語帳へ」遷移ボタン
- メインセクション
  - サーバー側でランダムに最大10件取得
  - 1問ずつ表示（フラッシュカード形式）
  - 進捗バー
  - 「答えを見る」→「次へ」→「最初からやり直す」のフロー
  - データ管理: `useState`（問題番号、答え表示状態）

### デザイン方針
- 余白を広めに取り、読みやすさを重視
- グラデーションやガラス風カードでモダン感を演出
- ダーク背景（`slate-950`）と明るいアクセントを組み合わせる
- 全画面で統一されたトーン＆マナー

### ナビゲーション
- 各画面のヘッダーに他画面への遷移ボタンを配置
- Next.jsの`Link`コンポーネントでSPA遷移

### データ永続化
- 現在: メモリ上（サーバー再起動でリセット）
- 将来: Prisma等のORMでDB接続予定
