### フェーズ5: クイズ画面（学習機能）
- `/quiz` ルートを作成
- Server Componentでランダムに10件の単語を取得（`app/quiz/page.tsx`）
- Client Componentでクイズのインタラクション実装（`app/quiz/QuizPlayer.tsx`）
  - 1問ずつ表示
  - 「答えを見る」ボタンで意味を表示
  - 「次へ」ボタンで次の問題に遷移
  - 進捗バー表示
- 全ページにクイズへのナビゲーションボタンを追加

### フェーズ6: Prisma + SQLite 導入
- Prisma v6 と SQLite をプロジェクトに導入
- `prisma/schema.prisma` に `Word` モデルを定義
  - `id`（自動採番）, `english`（英単語）, `japanese`（日本語の意味）, `createdAt`（作成日時）
- マイグレーション実行（`npx prisma migrate dev --name init`）
  - `prisma/dev.db` が作成される
  - `prisma/migrations/` にSQLマイグレーションファイルが生成
- `prisma/seed.ts` を作成し、TOEIC頻出単語10件の初期データを投入
  - `npx tsx prisma/seed.ts` または `npx prisma db seed` で実行
- `package.json` に `prisma.seed` 設定を追加
- `tsx` を devDependencies に追加（TypeScriptのシード実行用）

### フェーズ7: データベース連携（単語帳 + クイズ）
- `app/lib/prisma.ts` を作成（Prisma Client シングルトン）
  - 開発時のホットリロードで接続が増殖しないようにグローバルキャッシュ
- `app/vocabulary/actions.ts` をDB操作に書き換え
  - メモリ配列 → `prisma.word.create()` / `findMany()` / `delete()`
  - 単語帳アプリ（`/vocabulary`）がSQLiteと連携
  - クイズ画面（`/quiz`）も同じ `getVocabularies()` 経由でDBからデータ取得
- ビルドチェック成功（全ルート正常コンパイル）
